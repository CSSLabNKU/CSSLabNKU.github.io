### 计算机各类期刊、会议总结

<div class="miscellanea-content">
<strong><span style="color: #5040ae;">A. <a href="http://loccs.sjtu.edu.cn/typecho/index.php/archives/188/"><span style="color: #5040ae;">信息安全国际学术会议讨论</span></a></span><br/>
(Part of this blog is my personal opinion, while part of it comes from <a href="http://loccs.sjtu.edu.cn/typecho/index.php/category/conf/">http://loccs.sjtu.edu.cn/typecho/index.php/category/conf/</a>. 部分为个人见解，部分转自：<a href="http://loccs.sjtu.edu.cn/typecho/index.php/category/conf/">http://loccs.sjtu.edu.cn/typecho/index.php/category/conf/</a>，一介学生眼光，三五好友讨论结果，不妥不当之处难免，欢迎批评指正。)<br/>

<u>信息安全</u>是一个比较复杂的主题，包括了传统的密码学、安全攻击等内容，同时又包括了大量和操作系统、网络、计算机体系结构、硬件（不仅限于计算机，包括各类计算设备，智能卡等等），甚至金融、法律相关的内容，因此，在这些领域，同样有很多顶级学术会议值得我们关注。众所周知，期刊会议的档次无法代表论文的质量，只是在一流会议中出现一流成果的可能性要大一些。很多有开创性的一流成果没有发表在一流会议上，在一流会议中也不并少见长像好看、但实际没什么意义的文章。<strong>无论如何，了解会议的大致水平和特点，对初学者还是很有用的 (The true value of this list, we hope, mainly lies in providing some preliminary guidance for NEW guys in Information Security, and particularly, <span style="color: #5040ae;">for new students of our group</span>.</strong>)。
<br/>
首先是信息安全<strong>四大安全会议 + 三大密码会</strong>（美密，欧密，亚密）<br/>
<strong>S&P</strong> IEEE Symposium on Security and Privacy<br/>
每年11月截稿，第二年5月开会，永远在美国开会<br/>
<strong>CCS</strong> ACM Conference on Computer and Communications Security<br/>
每年5月截稿，同年10月开会，永远在美国开会<br/>
<strong>USENIX Security</strong> Usenix Security Symposium（System-oriented）<br/>
每年2月截稿，同年8月开会，永远在美国开会<br/>
<strong>NDSS</strong> ISOC Network and Distributed System Security（<u><a href="http://mp.weixin.qq.com/s?__biz=MzA3OTQ4OTY5OA==&amp;mid=2653145805&amp;idx=1&amp;sn=de43677e543279b16e7b0321b46481ed&amp;chksm=8465d500b3125c16d1ab81c500966927a2f17c2b3992fe1a3465425434378aefc2baad1e2668&amp;mpshare=1&amp;scene=23&amp;srcid=0113UQdRNSYuq8G0cyfBThqD#rd" target="_blank">大陆迄今录用最少</a></u>，<u><a href="http://www.cs.cornell.edu/andru/csconf.html" target="_blank">引用最好</a></u>）<br/>
每年8月截稿，第二年2月开会，永远在美国开会<br/>

<strong>CRYPTO</strong> International Cryptology Conference<br/>
美密会 每年2月截稿，同年8月开会，永远在美国开会<br/>
<strong>EUROCRYPT</strong> Annual International Conference on the Theory and Applications of Cryptographic Techniques<br/>
欧密会 每年10月截稿，翌年5月开会，永远在欧洲开会<br/>
<strong>ASIACRYPT</strong> Annual Int Conf on Theory and Application of Cryptology and Information Security （录用率比美密、欧密低）<br/>
亚密会 每年5月截稿，同年12月开会，永远在亚洲开会<br/>

接下来，根据<u><a href="http://jianying.space/conference-ranking.html" target="_blank">http://jianying.space/conference-ranking.html </a></u>和<u>https://mp.weixin.qq.com/s/Zx3CsLvxnLDFFlDc4B_sIw</u>的统计，我认为如下一些安全会议是很好的，刚好也是<strong>四个安全会+三个密码会</strong>：<br/>

<strong>ACSAC</strong> Annual Computer Security Applications Conference<br/>
<strong>ESORICS</strong> European Symposium on Research in Computer Security <br/>
<strong>CHES</strong> Annual Conference on Cryptographic Hardware and Embedded Systems<br/>
<strong>DSN</strong>  IFIP/IEEE International Conference on Dependable Systems and Networks<br/>

这4个会议目前每年的出席人数基本都稳定在200人以上，投稿量也比较大(200+)，说明大家都比较认可，有较强影响力，而且录用率也控制得很好：历年基本都在20%左右。其中ESORICS这个会永远在欧洲开，2017年投稿数达到330篇，创历史新高。<br/>

如果说上面这四个会偏向安全应用的话，下面这三个会则专注安全理论，里面文章的基础性、长期影响力来看比上面四个会甚至还要好一些：<br/>
<strong>CSF</strong> IEEE Computer Security Foundations Symposium<br/>
<strong>TCC</strong> IACR Theory of Cryptography Conference<br/>
<strong>PKC</strong> IACR International Conference on Practice and Theory of Public-Key Cryptography <br/>

介绍了上面14个会议，接下来的会议里面，排名最好的应该属于AsiaCCS、PETS和RAID。<br/>
<strong>AsiaCCS</strong> ACM Asia Conference on Information, Computer and Communications Security (2015年6月新更名为Aisa conference)<br/>
<strong>PETS</strong> Privacy Enhancing Technologies<br/>
<strong>RAID</strong> International Symposium on Research in Attacks, Intrusions and Defenses<br/>
<br/>
<strong>AsiaCCS 和 PETS</strong> 这两个会相对年轻，从与欧美同行朋友的了解情况看，<strong>AsiaCCS 和 PETS 貌似已稳步进入第二阵营</strong>。AsiaCCS是ACM近年在亚洲区力挺的一个会，看到一些大牛也在上面发文, 近两年的投稿量都在350+； PETS专注于隐私保护技术，随着人们对隐私的关注，圈子在快速扩大, 投稿量也快速上升，近两年投稿量300+，已显示很好的影响力。RAID 前几年投稿量低于100，近两年改名后（原名International Symposium on Recent Advances in Intrusion Detection）由USENIX出版，投稿量恢复到100以上。<br/>

接下来的安全会议在学术界<strong>比较知名</strong>（我见过的有名的安全研究人员大都愿意把在上面发表过的文章放在自己的主页上的）包括<br/>

<strong>FC</strong> International Conference on Financial Cryptography and Data Security<br/>
<strong>ACNS</strong> Applied Cryptography and Network Security<br/>
<strong>FSE </strong>Fast Software Encryption<br/>
<strong>SOUPS</strong> ACM Symposium on Usable Privacy and Security <br/>
<strong>WiSec</strong> ACM Conference on Security and Privacy in Wireless and Mobile Networks<br/>
<strong>DIMVA</strong> Detection of Intrusions and Malware and Vulnerability Assessment<br/>

这里面，FC由于密码货币的出现，近几年影响力上升很快；DIMVA在计算机安全攻防上很有影响力，但是由于圈子比较小，投稿篇数比较少(&lt;100), 看上去录用率高一些，RAID引用率要稍微好一些。FSE近年投稿量越来越少，2015年创新低76篇，可能与这个圈子太封闭有关系；ACNS和FC录用率控制得很好，这两个会更偏向密码学一些, 在系统安全方面相对接受的几率相对比较低, ACNS 开会地点离我们较近些，不过这两年ACNS也开始离开亚洲去US、UK开了；WiSec属于后起之秀，投稿量虽然不过百，但论文质量很高，国内目前中的不多。<br/>

下面四个安全会中，ISC和SEC属于比较综合型的会议，SecureComm和ICICS 更偏重系统安全、应用安全一些,。ICICS由大陆卿斯汉老师发起，是大陆信息安全走出去的一个难得窗口，文章虽多来自大陆，也有不少引用率很高的文章，2018年的投稿量超过200。<br/>
<strong>ISC</strong> Information Security Conference<br/>
<strong>SEC</strong> IFIP International  Information Security and Privacy Conference<br/>
<strong>SecureComm</strong> Security and Privacy for Emerging Areas in Communication Networks <br/>
<strong>ICICS</strong> International Conference on Information and Communications Security<br/>

下面这四个区域性召开的偏密码会议（也接收安全方向的论文），在密码学领域享有一定声誉，也不是很容易中，regular paper 为 LNCS 20页。ACISP是从澳洲发起的比较成功的一个会，每年在Australa 召开，过去有不少经典文章发表在这里，近两年投稿量超过150篇。中密会INSCRYPT近两年的投稿量在90篇左右，INDOCRYPT和ICISC近两年的投稿量在70篇左右。值得一提的是， INSCRYPT 是由信息安全国家重点实验室发起的在中国举办的密码年会。<br/>
<strong>ACISP </strong>Australasian Conference on Information Security and Privacy  澳密会<br/>
<strong>INSCRYPT </strong>International Conference on Information Security and Cryptology<strong> </strong>中密会<br/>
<strong>INDOCRYPT  </strong> International Conference on Cryptology in India  印密会<br/>
<strong>ICISC</strong> International Conference on Information Security and Cryptology 韩密会<br/>
<br/>
另外，<strong>近年有三个比较有潜力的会议面世</strong>，值得关注。尤其是IEEE EuroS&amp;P，这两届论文质量相当不错的说。<br/>
<strong> IEEE EuroS&amp;P</strong>  IEEE European Symposium on Security and Privacy  (http://www.ieee-security.org/TC/EuroSP2017/)<br/>
<strong> IEEE CNS</strong>  IEEE Conference on Communications and Network Security  (http://cns2017.ieee-cns.org/)<br/>
<strong> ACM CODASPY</strong> ACM Conference on Data and Application Security and Privacy<br/>
